## Tlias接口文档

1. ### 部门管理
   
   1. ##### 部门列表查询
      
      1. **基本信息**
         
         请求路径：`/department/lists`
         
         请求方式：`GET`
         
         接口描述：该接口用于部门列表全量数据查询
      
      2. **请求参数**
         
         无
      
      3. **响应数据**
         
         参数格式：`application/json`
         
         参数说明：
         
         | 参数名  | 类型           | 是否必须   | 备注              |
         |:----:|:------------:|:------:|:---------------:|
         | code | number       | 是      | 响应码，1代表成功，0代表失败 |
         | msg  | string       | 否      | 提示信息            |
         | data | object[]     | 否      | 返回的数据           |
         |      | - id         | number | 否               |
         |      | - name       | string | 否               |
         |      | - createTime | string | 否               |
         |      | - updateTime | string | 否               |
         
         响应数据样例：
         
         ```json
         {
             "code": 1,
             "msg": "成功",
             "data": [
                 {
                     "id": 1,
                     "name": "研发部",
                     "createTime": "2026-06-01T21:49:56",
                     "updateTime": "2026-06-01T21:49:56"
                 }
             ]
         }
         ```
   
   2. ##### 删除部门
      
      1. **基本信息**
         
         请求地址：`/department/list`
         
         请求方式：`DELETE`
         
         接口描述：该接口用于根据id值删除相应的部门列表信息
      
      2. **请求参数**
         
         参数格式：`queryString`
         
         参数说明：
         
         | 参数名 | 类型     | 是否必须 | 备注   |
         |:---:|:------:|:----:|:----:|
         | id  | number | 是    | 部门ID |
         
         参数样例：
         
         `/list?id=1`
         
         `/list?id=2`
      
      3. **响应数据**
         
         参数格式：`application/json`
         
         参数说明：
         
         | 参数名  | 类型       | 是否必须 | 备注              |
         |:----:|:--------:|:----:|:---------------:|
         | code | number   | 是    | 响应码，1代表成功，0代表失败 |
         | msg  | string   | 否    | 提示信息            |
         | data | object[] | 否    | 返回的数据           |
         
         参数样例：
         
         ```json
         {
             "code": 1,
             "msg": "成功",
             "data": null
         }
         ```

   4. ##### 部门数据查询

      2. **请求参数**

         请求参数说明：
         
         | 参数名 | 类型     | 是否必须 | 备注   |
         |:---:|:------:|:----:|:----:|
         | id  | number | 必须   | 部门ID |
         
         请求参数样例：
         
         `/list/1`
         
         `/list/2`
      
      3. **响应数据**

         |:----:|:--------:|:----:|:---------------:|
         | code | number   | 是    | 响应码，1代表成功，0代表失败 |
         | msg  | string   | 否    | 提示信息            |
         | data | object[] | 否    | 返回的数据           |
         
         参数样例：
         
         ```json
         {
             "code": 1,
             "msg": "成功",
             "data": [
                 {
                     "id": 1,
                     "name": "研发部",
                     "createTime": "2026-06-01T21:49:56",
                     "updateTime": "2026-06-01T21:49:56"
                 }
             ]
         }
         ```
   
   5. ##### 更改部门数据

         ```json

           "code": 1,

         }
         ```

2. ### 员工管理
   
   1. ##### 员工列表查询
      
      1. **基本信息**
         
         请求地址：`/employee/list`
         
         请求方式：`GET`
         
         接口描述：该接口用于查看员工信息
      
      2. **请求参数**
         
         参数格式：`queryString`
         
         参数说明：
         
         | 参数名称     | 类型     | 是否必须 | 示例  | 备注                     |
         |:--------:|:------:|:----:|:---:|:----------------------:|
         | page     | string | 是    | 1   | 分页查询的页码，如果未指定，默认为1     |
         | pageSize | stirng | 是    | 10  | 分页查询的每页记录数，如果未指定，默认为10 |
         
         参数样例：
         
         `/employee/list?page=1&pageSize=10`

         | 参数名称    | 类型       | 是否必须     | 备注              |
